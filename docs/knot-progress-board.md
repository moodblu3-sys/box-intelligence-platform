# Knot 進捗管理ボード

## 運用方針

Codexがヒアリングし、回答内容をもとに進捗・優先順位・次アクションを管理する。

進め方：

1. Codexが短い質問を出す
2. 回答をこのボードに反映する
3. P0 / P1 / P2に整理する
4. 次にやる作業を決める
5. 実装・資料作成・デモ確認を進める

## Jira自動起票デモ方針

| 項目 | 方針 |
|---|---|
| 目的 | Knowledge Deskが回答不能時に、問い合わせをJira Issueとして情シスへ渡せることを見せる |
| 実装方針 | Knowledge Desk APIからJira REST APIを呼び、デモ用ProjectへIssueを作成する |
| 安全策 | 環境変数でDRY_RUNを切り替える。デモ本番ではデモProjectにのみ起票する |
| 起票タイミング | `needsEscalation=true` の場合 |
| 当日見せる流れ | 質問 → 情報不足判定 → Jira起票 → Issue URL表示 |
| Jira Project Key | CIH |
| Jira Issue Type | Task想定。正式名称はAPI確認時に調整 |
| Jira認証情報 | `apps/knowledge-desk-api/.env` に保存。Gitには入れない |
| 実装状況 | Jira client、DRY_RUN、`jiraTicketUrl`返却、単体テスト、`smoke:jira`を追加済み |
| 確認状況 | `npm run test` 成功。`npm run smoke:jira` / `npm run smoke:jira:env` でdry-run成功。実Jira起票も確認済み |
| 未決 | 当日デモで実起票するか、DRY_RUN表示に留めるか |
