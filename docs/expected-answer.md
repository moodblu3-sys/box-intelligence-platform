# Expected Demo Answer

## Question

```text
日新テクノロジー株式会社向けに
Box AI活用基盤を提案したい。

Boxの議事録、
Teamsの議論、
Salesforceの商談情報、
SharePointの製品資料

を踏まえて、

* 提案方針
* リスク
* 次アクション

を整理してください。
```

## Expected Answer

日新テクノロジー株式会社には、Box AI単体ではなく、Boxを中心にTeams、SharePoint、Salesforceを横断するマルチソースRAG基盤として提案するのが妥当です。理由は、顧客課題がBox内文書の要約だけではなく、複数システムに分散した企業知識の探索、提案準備、ナレッジ共有にあるためです。

## 1. 提案方針

### 1.1 Boxを中心にした企業知識ハブとして提案

Boxには初回ヒアリング議事録や過去資料が保存されており、顧客はBox AIに関心を持っています。一方で、日新テクノロジー株式会社の実課題は、Box、Teams、SharePoint、Salesforceに情報が分散していることです。

そのため提案メッセージは、単なる「Box内文書のAI要約」ではなく、次のように置くべきです。

> Boxを中心に、Microsoft 365とSalesforceに分散した社内ナレッジを横断し、提案準備や問い合わせ対応に必要な情報を根拠付きで整理するAI活用基盤。

### 1.2 初期PoCは営業提案準備に絞る

初回から全社ナレッジ検索を狙うと、権限設計、データ品質、対象範囲が広がりすぎます。Teamsの議論でも、PoCは営業企画部門と情報システム部門に限定し、ユースケースは「営業提案準備」に絞る方針が妥当とされています。

PoCの対象は以下です。

| Source | PoCでの役割 |
|---|---|
| Box | 顧客ヒアリング議事録 |
| Teams | 営業・SEの温度感、勝ち筋、懸念点 |
| Salesforce | Account、Opportunity、Decision Makers、Amount、Timeline |
| SharePoint | Box AI機能、Enterprise Search設計、セキュリティFAQ |

### 1.3 顧客に刺さる価値

日新テクノロジー株式会社に対しては、次の価値を前面に出すべきです。

- 過去資料探索時間の削減
- 部門や個人に閉じたナレッジの共有
- 提案準備の初稿作成時間短縮
- Salesforceの商談文脈を踏まえた提案判断
- TeamsやSharePointにある実務上の文脈を含めた回答
- 根拠資料に戻れるAI回答

## 2. リスク

### 2.1 セキュリティと権限管理

最大のリスクは、権限外の資料が回答に混入することです。顧客の決裁者は、社内情報を勝手に学習されること、権限外資料が見えること、監査できないことを懸念しています。

対策:

- PoCは読み取り専用で開始する
- 対象ユーザーと対象部門を限定する
- 既存システムの権限を尊重する設計を説明する
- 参照ソースと回答ログを監査可能にする
- 機密資料をPoC対象に入れすぎない

### 2.2 Box AI単体との差別化不足

Box AI標準機能だけの説明では、Box内文書の要約に留まり、Salesforceの商談情報やTeamsの温度感を含めた提案判断には弱いです。顧客が本当に興味を持っているのは、Boxを含む社内情報全体の横断活用です。

対策:

- Box AIを補完する横断RAGとして位置付ける
- Box、Teams、SharePoint、Salesforceを引用した回答をデモで見せる
- 「単一文書要約」ではなく「提案判断支援」を訴求する

### 2.3 データ品質とスコープ肥大化

過去資料が重複していたり、古い資料が残っていたりすると、AI回答の品質が落ちる可能性があります。また、最初から全社展開すると権限設計と運用調整が重くなります。

対策:

- PoC対象データを営業提案準備に必要な範囲に限定する
- 古い資料や重複資料の扱いを事前に決める
- 成功指標を明確にする

## 3. 次アクション

### 3.1 次回提案で提示するもの

次回提案では、以下を提示するのがよいです。

1. Box中心のマルチソースRAG構成図
2. PoC対象ソースと対象ユーザー
3. セキュリティ・権限管理方針
4. 成功指標
5. 概算費用とスケジュール

### 3.2 推奨PoC計画

| 項目 | 内容 |
|---|---|
| 対象部門 | 営業企画部門、情報システム部門 |
| 対象ソース | Box、Teams、SharePoint、Salesforce |
| 対象ユースケース | 営業提案準備 |
| 期間 | 2026年7月開始、2026年9月に本番導入判断 |
| 成功指標 | 調査時間30%以上削減、提案初稿作成時間50%削減、根拠リンク付き回答 |
| 初期制約 | 読み取り専用、対象データ限定、対象ユーザー限定 |

### 3.3 顧客への確認事項

- PoC対象部門は営業企画と情シスで問題ないか
- 対象BoxフォルダとSharePointサイトをどこにするか
- Salesforce上のOpportunity項目をどこまでAI回答に含めるか
- Teamsのどのチャネルを対象にするか
- 権限検証の合格条件をどう定義するか

## 4. Recommended Positioning

日新テクノロジー株式会社への提案は、次の一文でまとめられます。

> Boxを入口に、Teams、SharePoint、Salesforceに分散した企業知識を安全に横断し、営業提案準備と社内ナレッジ活用を加速するAI基盤。

この方針であれば、顧客がすでに導入しているBox、Microsoft 365、Salesforceを活かしながら、生成AI活用への期待とセキュリティ懸念の両方に対応できます。
