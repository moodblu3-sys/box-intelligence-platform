# SharePoint Demo Pages

想定ソース: SharePoint  
サイト: Box Intelligence Platform Demo  
目的: 製品資料、設計メモ、セキュリティFAQをSharePointに分散配置し、Box / Teams / Salesforceと横断して参照させる。

---

## Page 1: Box AI標準機能まとめ

### 概要

Box AIは、Boxに保存された文書を対象に、要約、質問応答、文章生成を支援する機能である。Box内の契約書、議事録、提案書などを素早く理解する用途に適している。

### 主な機能

- Box内ファイルの要約
- 文書内容に対する質問応答
- 文書からのドラフト生成
- Box上のコンテンツを起点にしたナレッジ活用
- ファイルの権限を前提にした利用

### 日新テクノロジー株式会社への示唆

日新テクノロジー株式会社はBoxを導入済みであり、Box内の議事録や業務文書を活用する土台がある。初回ヒアリング議事録の要約や、Box上の文書確認にはBox AIの標準機能が有効である。

一方で、顧客課題はBox内文書だけでは完結しない。Teamsの営業・SE議論、SharePoint上の製品資料、Salesforceの商談情報にも重要な文脈が分散している。そのため、Box AI単体では「企業知識を横断して提案方針を整理する」用途には弱い。

### Box AI単体で弱いポイント

- Salesforceの商談フェーズ、金額、Decision Makersを直接参照できない
- Teams上の温度感や社内議論を含めた判断が難しい
- SharePointにある標準資料やセキュリティFAQとの横断整理が限定的
- 複数ソースの根拠をまとめて提案方針に落とし込むには追加基盤が必要

## Page 2: Enterprise Search設計メモ

### 設計方針

日新テクノロジー株式会社向けの提案では、Boxを中心にしたマルチソースRAG基盤を構成する。Boxを契約書・議事録・提案資料の中心リポジトリとしつつ、Teams、SharePoint、Salesforceを接続し、提案準備に必要な企業知識を横断検索できる状態を目指す。

### 対象ソース

| Source | 役割 |
|---|---|
| Box | 顧客ヒアリング議事録 |
| Teams | 営業・SEの検討ログ、顧客温度感、提案方針の議論 |
| SharePoint | 製品資料、標準提案テンプレート、セキュリティFAQ |
| Salesforce | Account情報、Opportunity情報、決裁者、金額、タイムライン |

### PoCスコープ

初期PoCでは全社展開を避け、営業企画部門と情報システム部門に限定する。対象ユースケースは「営業提案準備」に絞る。ユーザーは過去資料を探す代わりに、自然文で顧客名と目的を入力し、提案方針、リスク、次アクションを取得する。

### 成功指標

- 提案準備に必要な調査時間を30%以上削減
- 提案初稿作成時間を50%削減
- 回答に根拠資料リンクが付与される
- 権限外の資料が回答に混入しない
- 営業とSEが同じ前提情報を参照できる

### 推奨アーキテクチャ

1. 各SaaSから読み取り専用でデータを取得する。
2. 文書、チャット、商談情報をDocumentとしてindexする。
3. ソース、更新日時、所有者、権限情報をmetadataとして保持する。
4. ユーザー質問に対して関連文書を検索する。
5. LLMに検索結果を渡し、根拠付きで提案方針、リスク、次アクションを生成する。

## Page 3: セキュリティ・権限管理FAQ

### Q1. 生成AIが社内情報を勝手に学習しないか。

PoCでは、接続先データを外部モデルの学習に利用しない設定を前提とする。LLMに渡す情報は検索で取得された必要範囲の文脈に限定する。利用するLLMプロバイダーのデータ保持ポリシーも提案時に明示する。

### Q2. 権限外の資料が回答に出ないか。

本番導入時は、既存システムのアクセス権限を尊重する構成を採用する。Box、Teams、SharePoint、Salesforceの権限情報を参照し、ユーザーがアクセス可能な情報だけを検索・回答対象にする。PoCでは対象部門と対象データを限定し、権限検証を行う。

### Q3. 監査ログは残せるか。

誰が、いつ、どの質問を行い、どのソースを参照したかを記録する方針とする。日新テクノロジー株式会社のセキュリティレビューでは、回答内容だけでなく参照ソースの追跡可能性が重要である。

### Q4. PoCで最初に制限すべきことは何か。

最初から全社データを対象にしない。営業企画部門と情報システム部門に限定し、Boxの議事録、Teamsの提案議論、SharePointの標準資料、Salesforceの対象商談だけを接続する。

### Q5. セキュリティ面の主なリスクは何か。

- 権限設計が不十分なまま全社展開すること
- 古い資料や重複資料をAIが根拠にすること
- 回答の根拠資料が確認できないこと
- 機密情報を含むデータをPoC対象に含めすぎること
- Salesforce上の商談情報と文書情報の更新タイミングがずれること

### Q6. 日新テクノロジー株式会社への推奨回答

最初は読み取り専用のPoCとして開始する。対象ソースとユーザーを限定し、根拠リンク付き回答、アクセス制御、監査ログを確認する。PoCで効果が確認できた後に、対象部門とデータソースを段階的に拡張する。
