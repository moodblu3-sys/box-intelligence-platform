# Knot デモ準備棚卸しシート

## 目的

Knotの開発状況と面接デモ準備状況を整理するための記入シート。

面接では、Knotを「Onyx OSSをベースにした社内ナレッジ問い合わせAI」として見せる。
このシートでは、実装済み機能、動作確認状況、デモで使う範囲、リスク、次アクションを整理する。

## 記入ルール

ステータスは以下から選ぶ。

| ステータス | 意味 |
|---|---|
| OK | デモでそのまま使える |
| 要確認 | 使える可能性はあるが、再確認が必要 |
| 要修正 | デモ前に直す必要がある |
| 見せない | 実装済みでも今回のデモ対象外 |
| 未着手 | まだ作っていない |

優先度は以下から選ぶ。

| 優先度 | 意味 |
|---|---|
| P0 | デモ成立に必須 |
| P1 | あると説得力が上がる |
| P2 | 時間があれば対応 |
